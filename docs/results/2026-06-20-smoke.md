# Tiny Shakespeare Smoke Result

## Protocol

The first end-to-end check used `configs/smoke.toml`: one Transformer layer, 32-wide
embeddings, context 32, batch size 8, 40 CPU steps, pressure threshold 4, and seed 1337.
Both arms used the pinned Tiny Shakespeare corpus and matched schedules.

Command:

```bash
UV_CACHE_DIR=/games/ailab/.uv-cache uv run lat compare \
  --config configs/smoke.toml \
  --dataset-path data/huggingface/datasets--SamPIngram--tinyshakespeare/snapshots/6d8bc3fdfca13bf8a128bb0e0914cead1e2d208c/input.txt \
  --output runs/smoke
```

## Observations

| Arm | Initial validation | Final validation | Final train | Moves at logged steps | Final saturation | Final zeros |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| Quinary | 4.2296 | 2.8059 | 2.7598 | 2,914 | 45.77% | 17.95% |
| Septenary | 4.2075 | 2.8254 | 2.8000 | 3,227 | 35.83% | 13.17% |

Both arms passed the first viability check: their validation losses moved below the roughly
`ln(65)` random-character reference and their integer codes moved substantially. Quinary
saturated more quickly in this short run. One seed and 40 steps are not enough to rank the
methods; the three-seed 2,000-step configuration remains the comparative experiment.

Generated CSV, safetensors, JSON, and PNG artifacts are under ignored `runs/smoke/` in the
verification checkout. Re-run the command above to reproduce them in another checkout.
